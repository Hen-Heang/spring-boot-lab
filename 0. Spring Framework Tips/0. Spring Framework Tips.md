## Section 1 — Java Fundamentals Tips 🔥

### OOP
- Always prefer **composition over inheritance** — Spring itself uses this pattern heavily
- Use `interface` to define contracts, `abstract class` for shared behavior with state
- Encapsulation tip: never expose fields directly — always use getters/setters or Records

```java
// BAD
public String name;

// GOOD
private String name;
public String getName() { return name; }

// BEST (2026) — use Record for pure data carriers
public record UserDto(String name, String email) {}
```

### Collections 🔥
- Use `List.of()`, `Map.of()`, `Set.of()` for immutable collections (Java 9+)
- Prefer `Map<String, Object>` with `HashMap` for general use, `LinkedHashMap` when order matters
- Never modify a `List.of()` — it throws `UnsupportedOperationException`

```java
// Common interview trap
List<String> list = List.of("a", "b");
list.add("c"); // throws UnsupportedOperationException!

// Safe mutable copy
List<String> mutable = new ArrayList<>(List.of("a", "b"));
```

### Exception Handling
- Always catch specific exceptions, not just `Exception`
- Create custom exceptions for your domain — companies look for this

```java
// Custom exception pattern (used in every real project)
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(Long id) {
        super("User not found with id: " + id);
    }
}
```

### Streams + Lambda 🔥
- Master these 5 operations first: `filter`, `map`, `collect`, `findFirst`, `anyMatch`
- Use `Collectors.groupingBy()` — very common in reporting APIs

```java
// Filter + map + collect — the most common pattern
List<String> activeEmails = users.stream()
    .filter(u -> u.isActive())
    .map(User::getEmail)
    .collect(Collectors.toList());

// GroupingBy — useful for reports
Map<String, List<User>> byRole = users.stream()
    .collect(Collectors.groupingBy(User::getRole));
```

### Virtual Threads (Project Loom) ✅
- Available from Java 21 — enable in Spring Boot 3.2+ with one property
- Use for I/O-heavy tasks (DB calls, HTTP calls) — don't use for CPU-heavy tasks

```yaml
# application.yml — enable virtual threads in Spring Boot 3.2+
spring:
  threads:
    virtual:
      enabled: true
```

### Records ✅
- Perfect for DTOs — replaces boilerplate getters/setters/constructors
- Records are immutable by default — great for request/response objects

```java
// DTO with Record — clean and modern
public record CreateUserRequest(
    @NotBlank String name,
    @Email String email,
    @NotBlank String password
) {}

public record UserResponse(Long id, String name, String email) {}
```

### Optional
- Use `Optional` as return type for methods that might return null
- Never use `Optional.get()` without `isPresent()` check — use `orElseThrow()` instead

```java
// BAD
User user = userRepository.findById(id).get(); // NullPointerException risk!

// GOOD
User user = userRepository.findById(id)
    .orElseThrow(() -> new UserNotFoundException(id));
```

## Section 2 — Spring Core Tips ✅

### IoC / DI
- **Constructor injection is always preferred** over field injection (`@Autowired` on field)
- Reason: easier to test, dependencies are explicit, supports immutability

```java
// BAD — field injection (hard to test, hides dependencies)
@Service
public class UserService {
    @Autowired
    private UserRepository userRepository;
}

// GOOD — constructor injection
@Service
@RequiredArgsConstructor  // Lombok generates constructor
public class UserService {
    private final UserRepository userRepository;
}
```

### Bean Scopes
- `@Scope("singleton")` — default, one instance per Spring container
- `@Scope("prototype")` — new instance every time it's injected
- 99% of the time you use singleton — don't over-think it

### @Configuration vs @Component
- `@Configuration` — for bean factory classes (creates `@Bean` methods)
- `@Component` → `@Service` → `@Repository` — for your own classes
- `@Repository` adds exception translation (converts DB exceptions to Spring exceptions)

```java
@Configuration
public class AppConfig {
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

### Profiles
- Use `@Profile("dev")` and `@Profile("prod")` to swap beans by environment
- Very useful for switching between in-memory DB (dev) and real DB (prod)

```java
@Configuration
@Profile("dev")
public class DevDataSourceConfig {
    @Bean
    public DataSource dataSource() {
        return new EmbeddedDatabaseBuilder()
            .setType(EmbeddedDatabaseType.H2).build();
    }
}
```

## Section 3 — Spring Boot Tips 🔥

### Project Structure — Follow This Layer Pattern
```
src/main/java/com/yourapp/
├── controller/       ← HTTP layer (receive request, return response)
├── service/          ← Business logic
├── repository/       ← DB access
├── entity/           ← JPA entities (DB table mapping)
├── dto/              ← Request/Response objects (never expose entity directly!)
├── exception/        ← Custom exceptions + GlobalExceptionHandler
├── config/           ← Security, Swagger, Bean configs
└── util/             ← Shared helper methods
```

> Never return `Entity` directly from controller — always map to DTO first.

### Global Exception Handler 🔥
- Every real project needs this. Set it up on Day 1.

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex) {
        return ResponseEntity.status(HttpStatus.NOT_FOUND)
            .body(new ErrorResponse(404, "NOT_FOUND", ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidation(MethodArgumentNotValidException ex) {
        String message = ex.getBindingResult().getFieldErrors()
            .stream()
            .map(e -> e.getField() + ": " + e.getDefaultMessage())
            .collect(Collectors.joining(", "));
        return ResponseEntity.badRequest()
            .body(new ErrorResponse(400, "INVALID_INPUT", message));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGeneral(Exception ex) {
        return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(new ErrorResponse(500, "INTERNAL_ERROR", "Something went wrong"));
    }
}
```

### Logging with Logback
- Log at `INFO` level in production, `DEBUG` in development
- Always include context (userId, requestId) in logs

```java
@Slf4j  // Lombok — adds private static final Logger log = ...
@Service
public class UserService {

    public UserResponse getUser(Long id) {
        log.info("Fetching user with id={}", id);    // use {} not + concatenation
        User user = userRepository.findById(id)
            .orElseThrow(() -> {
                log.warn("User not found: id={}", id);
                return new UserNotFoundException(id);
            });
        log.debug("User found: {}", user.getEmail());
        return mapToResponse(user);
    }
}
```

## Section 4 — Database Tips 🔥

### SQL Tips for Interviews
```sql
-- Most common interview patterns

-- 1. Find duplicates
SELECT email, COUNT(*) FROM users GROUP BY email HAVING COUNT(*) > 1;

-- 2. Get top N per group
SELECT * FROM (
    SELECT *, ROW_NUMBER() OVER (PARTITION BY category ORDER BY price DESC) AS rn
    FROM products
) t WHERE t.rn <= 3;

-- 3. Self JOIN — find users with same manager
SELECT a.name, b.name AS manager
FROM employees a
JOIN employees b ON a.manager_id = b.id;
```

### JPA Tips 🔥
- Use `@Column(nullable = false)` to enforce DB constraints
- Always define `equals()` and `hashCode()` on entities using `@Id` only
- Use `FetchType.LAZY` by default — avoid loading data you don't need

```java
@Entity
@Table(name = "users")
@Getter
@NoArgsConstructor(access = AccessLevel.PROTECTED)  // JPA needs no-arg constructor
public class User {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true)
    private String email;

    @OneToMany(mappedBy = "user", fetch = FetchType.LAZY, cascade = CascadeType.ALL)
    private List<Order> orders = new ArrayList<>();
}
```

### N+1 Problem — Very Common Interview Question 🔥
```java
// BAD — causes N+1 queries
List<User> users = userRepository.findAll();
users.forEach(u -> u.getOrders().size()); // fires a query PER user!

// GOOD — use JOIN FETCH
@Query("SELECT u FROM User u JOIN FETCH u.orders WHERE u.id = :id")
Optional<User> findByIdWithOrders(@Param("id") Long id);
```

### MyBatis Tips ✅ (Important for Korea)
- Use `@Mapper` interface + XML for complex queries
- `@Select`, `@Insert`, `@Update`, `@Delete` for simple operations

```java
@Mapper
public interface UserMapper {

    @Select("SELECT * FROM users WHERE id = #{id}")
    User findById(Long id);

    @Insert("INSERT INTO users(name, email) VALUES(#{name}, #{email})")
    @Options(useGeneratedKeys = true, keyProperty = "id")
    void insert(User user);
}
```

```xml
<!-- UserMapper.xml — use for complex dynamic SQL -->
<mapper namespace="com.example.mapper.UserMapper">
    <select id="findByCondition" resultType="User">
        SELECT * FROM users
        <where>
            <if test="name != null">AND name LIKE CONCAT('%', #{name}, '%')</if>
            <if test="role != null">AND role = #{role}</if>
        </where>
        ORDER BY created_at DESC
        LIMIT #{size} OFFSET #{offset}
    </select>
</mapper>
```

### Transaction Management
- `@Transactional` on service methods, not repository
- Read-only queries should use `@Transactional(readOnly = true)` — better performance

```java
@Service
@RequiredArgsConstructor
public class OrderService {

    @Transactional  // write operation — full transaction
    public OrderResponse createOrder(CreateOrderRequest request) {
        // if any step fails, entire operation rolls back
        Order order = orderRepository.save(mapToEntity(request));
        inventoryService.deductStock(request.productId(), request.quantity());
        return mapToResponse(order);
    }

    @Transactional(readOnly = true)  // read-only — better performance
    public Page<OrderResponse> getOrders(Pageable pageable) {
        return orderRepository.findAll(pageable).map(this::mapToResponse);
    }
}
```

## Section 5 — Spring Security Tips 🔥

### JWT Setup Pattern
```
Client → POST /api/auth/login (email + password)
       ← 200 OK { accessToken, refreshToken }

Client → GET /api/users (Authorization: Bearer <accessToken>)
       ← 200 OK [...]

Client → POST /api/auth/refresh (refreshToken)
       ← 200 OK { new accessToken }
```

### Security Config (Spring Boot 3.x)
```java
@Configuration
@EnableWebSecurity
@RequiredArgsConstructor
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(session ->
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**", "/swagger-ui/**", "/v3/api-docs/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

### JWT Filter
```java
@Component
@RequiredArgsConstructor
public class JwtAuthenticationFilter extends OncePerRequestFilter {

    private final JwtService jwtService;
    private final UserDetailsService userDetailsService;

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        String authHeader = request.getHeader("Authorization");
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            filterChain.doFilter(request, response);
            return;
        }

        String token = authHeader.substring(7);
        String username = jwtService.extractUsername(token);

        if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {
            UserDetails userDetails = userDetailsService.loadUserByUsername(username);
            if (jwtService.isTokenValid(token, userDetails)) {
                UsernamePasswordAuthenticationToken authToken =
                    new UsernamePasswordAuthenticationToken(userDetails, null, userDetails.getAuthorities());
                authToken.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
                SecurityContextHolder.getContext().setAuthentication(authToken);
            }
        }
        filterChain.doFilter(request, response);
    }
}
```

## Section 6 — Testing Tips ✅

### Unit Test — Service Layer
```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {

    @Mock
    private UserRepository userRepository;

    @InjectMocks
    private UserService userService;

    @Test
    void getUser_whenExists_returnsResponse() {
        // Arrange
        User user = new User(1L, "Hen Heang", "hen@example.com");
        when(userRepository.findById(1L)).thenReturn(Optional.of(user));

        // Act
        UserResponse response = userService.getUser(1L);

        // Assert
        assertThat(response.name()).isEqualTo("Hen Heang");
        verify(userRepository, times(1)).findById(1L);
    }

    @Test
    void getUser_whenNotFound_throwsException() {
        when(userRepository.findById(99L)).thenReturn(Optional.empty());
        assertThatThrownBy(() -> userService.getUser(99L))
            .isInstanceOf(UserNotFoundException.class);
    }
}
```

## Section 7 — REST API Design Tips 🔥

### URI Naming — Do This Right
```
# BAD — verb in URL, not RESTful
GET  /getUsers
POST /createUser
POST /deleteUser/1

# GOOD — noun + HTTP verb
GET    /api/v1/users          → list all users
POST   /api/v1/users          → create user
GET    /api/v1/users/{id}     → get one user
PUT    /api/v1/users/{id}     → full update
PATCH  /api/v1/users/{id}     → partial update
DELETE /api/v1/users/{id}     → delete user

# Nested resource
GET    /api/v1/users/{id}/orders      → get user's orders
POST   /api/v1/users/{id}/orders      → create order for user
```

### Standard Error Response Format 🔥
```java
// Always use a consistent error format — companies check this
public record ErrorResponse(
    int status,
    String code,
    String message,
    LocalDateTime timestamp
) {
    public static ErrorResponse of(int status, String code, String message) {
        return new ErrorResponse(status, code, message, LocalDateTime.now());
    }
}
```

### Standard Pagination Response
```java
public record PageResponse<T>(
    List<T> content,
    int page,
    int size,
    long totalElements,
    int totalPages,
    boolean last
) {
    public static <T> PageResponse<T> from(Page<T> page) {
        return new PageResponse<>(
            page.getContent(),
            page.getNumber(),
            page.getSize(),
            page.getTotalElements(),
            page.getTotalPages(),
            page.isLast()
        );
    }
}
```

### HTTP Status Codes — Know These Cold 🔥
| Code | When to use |
|---|---|
| 200 OK | Successful GET, PUT, PATCH |
| 201 Created | Successful POST (return created resource) |
| 204 No Content | Successful DELETE |
| 400 Bad Request | Invalid input / validation failure |
| 401 Unauthorized | Not logged in (no/invalid token) |
| 403 Forbidden | Logged in but not allowed (wrong role) |
| 404 Not Found | Resource doesn't exist |
| 409 Conflict | Duplicate data (email already exists) |
| 500 Internal Server Error | Unexpected server error |

### MDC — Add traceId to Every Log Line
```java
@Component
public class LoggingFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain chain) throws ServletException, IOException {
        String traceId = UUID.randomUUID().toString().substring(0, 8);
        MDC.put("traceId", traceId);
        try {
            chain.doFilter(request, response);
        } finally {
            MDC.clear();
        }
    }
}
```

```xml
<!-- Add %X{traceId} to your log pattern -->
<pattern>%d{HH:mm:ss} [traceId=%X{traceId}] %-5level %logger - %msg%n</pattern>
```

## Quick Reference — Most Used Annotations

| Annotation | Where | Purpose |
|---|---|---|
| `@RestController` | Class | Controller that returns JSON |
| `@RequestMapping` | Class | Base URL for all endpoints |
| `@GetMapping` / `@PostMapping` | Method | Map HTTP methods |
| `@PathVariable` | Param | Extract `{id}` from URL |
| `@RequestParam` | Param | Extract `?page=0` from query string |
| `@RequestBody` | Param | Read JSON request body |
| `@Valid` | Param | Trigger bean validation |
| `@Service` | Class | Business logic layer |
| `@Repository` | Class | Data access layer |
| `@Transactional` | Method/Class | Wrap in DB transaction |
| `@Slf4j` | Class | Inject `log` logger (Lombok) |
| `@RequiredArgsConstructor` | Class | Constructor injection (Lombok) |
| `@PreAuthorize` | Method | Role-based access control |
| `@Profile` | Class | Load bean only in specific profile |
| `@Value` | Field | Inject value from `application.yml` |
| `@Scheduled` | Method | Run method on a schedule |
